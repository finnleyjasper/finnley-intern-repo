# When to use AI?
- AI should be used for answering questions requiring nuance that regular googling may not take into account
- AI can aid productivity in development for completing simple tasks the users understands how to do, but would be quicker for AI to generate *ie. Creating a simple readable table in HTML with CSS styling*
- A user's own skills should be primarily relied upon, particularly when learning new skills to ensure real learning is taking place and there is an understanding of core concepts

# How to avoid over-reliance?
- I will ensure that I understand the core concepts and methodology of any task I offload to AI - *ie. I know how to use HTML and CSS to create a readable table with basic styling, though this is a menial task that by using AI, I would save time without sacrificing my understanding*
- If turning to AI for a solution to a problem, I will critically analyse the solution generated and compare to my own ideas as well as my teammates if applicable

# When not to use?
- The AI should never be allowed access to confidential or important data such as the personal information of clients or passwords
- I will ensure that the information or examples I feed to AI **do not** include any senstitive information, and confirm that this data is not freely availible
