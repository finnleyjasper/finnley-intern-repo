# Reflection
### Top Shortcuts
- ```Cmd + P``` to quickly open files by name
- ```Opt + Cmd + S``` to save all open files
- ```Cmd + F ``` with ```Cmd + H``` to find and replace in the current file
- ```Cmd + C`` with ```Cmd + V`` to copy and paste
- ```Cmd + Z``` to undo the last action
- ```Cmd + Shift + Z``` to redo the last action
- ```Cmd + /``` to comment or uncomment a line of code
- ```Shift + TAB``` to un-indent a line of code
- ```Shift + Ctrl + A``` to select a block of code
- ```Cmd + Shift + O``` to quickly navigate to a symbol in the current file

### Which shortcuts do you think will be most useful to you?
I think copy, paste and undo will my most used shortcuts. However I think specific to VSCode, cmd + shift + s is very useful. Particularly when working in an engine like unity, where many scripts with complicated dependencies can be open and used at once. When making changes, Unity will only recognise changes after the files have been saved, so being able to save everything open at once saves a lot of debugging time.

### How can shortcuts improve your productivity?
Shortcuts can save a lot of time when working on a project by reducing the need to switch between mouse and keyboard or navigating through menus. It is much quicker to hit a few keys than to spend time clicking around menus if you don't need to.
