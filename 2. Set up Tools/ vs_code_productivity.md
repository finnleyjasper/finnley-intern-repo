# Reflection
## Top Shortcuts
### Navigation
- ```Cmd + P``` to quickly open files by name
- ```CMd + 2``` to open a second window
- ```Cmd + Shift + O``` to quickly navigate to a symbol in the current file
### Editing
- ```Opt + Cmd + S``` to save all open files
- ```Cmd + F ``` with ```Cmd + H``` to find and replace in the current file
- ```Cmd + /``` to comment or uncomment a line of code
- ```Shift + TAB``` to un-indent a line of code
- ```Option + Cmd + Up``` to add cursor above & ```Option + Cmd + Down``` to add cursor below for multiple cursors
### Debugging
- ```Shift + D``` to run and debug
### Terminal Commands
- ```Cmd + K``` toclear the terminal

### Which shortcuts do you think will be most useful to you?
I think copy, paste and undo will my most used shortcuts. However I think specific to VSCode, cmd + shift + s is very useful. Particularly when working in an engine like unity, where many scripts with complicated dependencies can be open and used at once. When making changes, Unity will only recognise changes after the files have been saved, so being able to save everything open at once saves a lot of debugging time. Additionally, the shortcuts for having multiple cursors has been very time-saving; I did not know about or use these previously, and they have been great for my productivity.

### How can shortcuts improve your productivity?
Shortcuts can save a lot of time when working on a project by reducing the need to switch between mouse and keyboard or navigating through menus. It is much quicker to hit a few keys than to spend time clicking around menus if you don't need to. For example, when formatting my responces for these tasks, it is much quicker to use multiple cursors to add markdown formatting (such as indicating titles with #), rather than moving the cursor with my mouse and re-typing the syntax.
