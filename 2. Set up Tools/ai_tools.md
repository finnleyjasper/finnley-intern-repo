# Reflection
### Which AI tools did you try?
I tried out Github Copilot and ChatGPT for coding.

### What worked well? What didn’t?
I have found Github Copilot to be pretty useful for generating basic code and for inline suggestions, though sometimes it does get the most basic things wrong - for example, generating the template for a table in markdown and putting it at the top of my file when I requested it to be placed at the bottom. However, overall it is a huge time-saver and quite reliable, so long as you know enough to recognise when it makes mistakes.

I have also found ChatGPT to be very useful, particularly when debugging as I can get effective explanations of what went wrong instead of just the answers, which has been great for my learning. It is also useful when generating code, though though as it's not integrated like Copilot, it is less effective for larger projects with complex codebases.

### When do you think AI is most useful for coding?
I think AI is most useful when you are coding something that uses skills you are familiar with, but the task you need to do is tedious or time-consuming; types of tasks that take a lot of time but don't serve as a learning opportunity. For example, I know how to create a form using HTML and PHP to send a user's input to a connected database. However, it's been a while sinve I've done this, and I would need to double check syntax and the refresh my memory. Here, I believe using AI to produce this code would save me a huge amount of time, as I can gain the same memory-refresh from reading over the generated code, without having to spend time reading other resources and coding it myself.

I also think it can be very useful for debugging, as I can get explanations of what went wrong and how to fix it - especially if the model has the context of your whole project. Resources like Stack Overflow and other sites are fantastic, though they can't take in the nuance of your specific workspace, so here AI can be very useful.
