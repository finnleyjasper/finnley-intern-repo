# Reflection
### Have you used Git before? If so, in what context?
Yes, I have used Git a lot before, in all my previous team projects at university. I have mostly used it with games projects, so handling C# files with Unity. I have experience with pushing, pulling and resolving merge conflicts, as well as branching and merging. Conflicts caused by Unity files such as scenes are difficult to solve, so I have had a lot of experience working through complex merge conflicts such as these.

### Which Git client (if any) did you choose? Why?
I have used Github Desktop primarily, but I have also used BitBucket before. I chose Github Desktop as I find the visual interface much easier to understand than the command line or VS Code's options. I had to use Bitbucket for a specific unit I studies, and I thought Bitbucket's interface was more confusing, and learning it did not offer any benefits over Github Desktop, so I have stuck with that.

### What was the most interesting thing you learned about Git today?
Today I did spend some time exploring in-editor Git options in VS Code, and I found it useful to learn you can stage and commit changes from there. Github Desktop is my go-to for source control, so I had not explored the options in VS Code before. I think it is useful to know how to do this from the editor, as it is more efficient than switching between applications, but I still prefer the visual interface of Github Desktop for handling merge conflicts and branching as I am familar with it.
